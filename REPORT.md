# LeadKeeper — Отчёт о выполнении

## Что сделано

1. **Fullstack-приложение** с React + TypeScript (frontend) и FastAPI + SQLite (backend)
2. **Форма заявки** с отдельными полями: имя, телефон, email, компания, комментарий, согласие
3. **Строгая валидация**:
   - Телефон: `+КодСтраны Номер` (максимум 16 цифр после +)
   - Email: RFC 5322 compliant
   - **Хотя бы один контакт (phone или email) обязателен**
   - Валидация показывается только при отправке (не в реальном времени)
4. **SQLite база данных**: заявки сохраняются локально
5. **Email-уведомления** через SMTP:
   - HTML-шаблоны (PostHog-styled)
   - Настройка через `.env` файл
   - Поддержка SSL для порта 465
6. **Служебная страница `/admin`**: таблица с phone/email колонками
7. **PostHog-inspired дизайн**: кремовый фон, жёлто-оранжевый акцент
8. **Makefile** для Linux/macOS + **make.bat** для Windows
9. **API документация**: Swagger UI (`/docs`) + ReDoc (`/redoc`)
10. **README.md**: полная документация, чек-лист, валидация, email

## Исправленные проблемы

### 1. Валидация контакта

- Теперь работает правило "хотя бы один контакт обязателен"
- Ошибка показывается только при отправке, не при вводе
- Пользователь может спокойно вводить и исправлять данные

### 2. SMTP отправка

- Исправлена логика для SSL (порт 465 использует SMTP_SSL)
- Добавлено логирование для диагностики
- Требуется **пароль приложения**, не обычный пароль

### 3. UX валидации

- Ошибки полей показываются только при отправке
- При потере фокуса поля — валидация формата (если поле заполнено)
- Динамическая метка "\*" — обязательно только если другой контакт не заполнен

## Валидация

### Телефон

```
+7 999 123-45-67  ✓
+44 20 7946 0958  ✓
+1 (555) 123-4567 ✓
1234567890        ✓ (без +)
+12345678901234567 ✗ (слишком длинный)
```

### Email

```
ivan@example.com        ✓
user.name@company.co.uk ✓
user_name@mail.ru       ✓
user..name@mail.ru      ✗ (две точки)
.user@mail.ru           ✗ (начинается с точки)
user.@mail.ru           ✗ (заканчивается точкой)
```

### Обязательно

- Имя (2-100 символов)
- Телефон **ИЛИ** email (хотя бы один)
- Согласие на обработку данных

## Email-уведомления

Для работы SMTP нужно настроить `.env`:

```env
OWNER_EMAIL=test@d9911.org
SMTP_HOST=smtp.mail.ru
SMTP_USER=d.9911@mail.ru
SMTP_PASS=ПАРОЛЬ_ПРИЛОЖЕНИЯ  # Не обычный пароль!
SMTP_PORT=465
SMTP_SECURE=true
SMTP_METHOD=1
```

**Важно:** Для Mail.ru нужен "пароль приложения", не обычный пароль.

## Не делал намеренно

- Авторизация `/admin` — указано в requirements
- Интеграция с CRM, Telegram
- Docker / сложная инфраструктура
- Сложный дизайн (только аккуратный PostHog-inspired UI)
- Production-миграции, логирование, мониторинг
- Защита от спама (rate limiting / CAPTCHA)

## Допущения

1. **SQLite** — подходит для прототипа, для production нужен PostgreSQL
2. **Валидация контакта** — простая (email с @ или телефон с цифрами)
3. **Без защиты от спама** — в production нужны rate limiting или captcha
4. **Согласие** — чекбокс, в production нужна юридически корректная форма

## Production-риски

- `/admin` открыт → добавить JWT
- Нет защиты от спама → CAPTCHA
- SQLite → PostgreSQL для масштабирования
- Email → пароль приложения обязателен
