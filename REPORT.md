# LeadKeeper — Отчёт о выполнении

## Что сделано

1. **Fullstack-приложение** с React + TypeScript (frontend) и FastAPI + SQLite (backend)
2. **Форма заявки** с полями: имя, контакт (email/телефон), компания, комментарий, согласие
3. **Клиентская валидация**: проверка обязательных полей, формат контакта, чекбокс согласия
4. **Серверная валидация**: дублирование проверок на backend для безопасности
5. **SQLite база данных**: заявки сохраняются локально
6. **Служебная страница /admin**: таблица всех заявок с датой, именем, контактом, компанией, комментарием
7. **Состояния UI**: idle, loading, success, error — понятные пользователю
8. **Адаптивный дизайн**: карточки, таблица, мобильная верстка
9. **Makefile**: команды `make install`, `make dev`, `make backend`, `make frontend`, `make clean`
10. **README.md**: инструкции запуска, чек-лист проверки, допущения, production-риски

## Скриншоты

Форма заявки и страница админки описаны в README.md.

## Не делал намеренно (согласно заданию)

- Авторизация `/admin` — указано в requirements
- Интеграция с CRM, Telegram, email
- Docker / сложная инфраструктура
- Сложный дизайн (только аккуратный UI)
- Production-миграции, логирование, мониторинг

## Допущения

1. **SQLite** — подходит для прототипа, для production нужен PostgreSQL
2. **Валидация контакта** — простая (email с @ или телефон с цифрами)
3. **Без защиты от спама** — в production нужны rate limiting или captcha
4. **Согласие** — чекбокс, в production нужна юридически корректная форма

## Production-риски

- `/admin` открыт без авторизации → закрыть VPN/паролем или добавить JWT
- Нет защиты от спама → добавить rate limiting + CAPTCHA
- SQLite не масштабируется → миграция на PostgreSQL
- Нет миграций БД → добавить Alembic
- Согласие на обработку данных требует юридического оформления (152-ФЗ, GDPR)
